🛠️ Hardware & Software Integration Post-Mortem

A precise log of core architectural vulnerabilities caught, diagnosed, and corrected during the 3-week development lifecycle.

💻 Java Application Suite Failures

Project: The Haunted Waltz 5.0 — The "Forever Right" Tracking Glitch

Vulnerability: Duplicate logic incrementation within the vector calculus pipeline.

if (ghostX < playerX) nextX++;
else if (ghostX > playerX) nextX++; // <-- The Bug

System Failure: Instead of decrementing when the target was to its left, the asset incremented regardless, causing it to flee from the player and run into a permanent boundary lock.

Resolution: Refactored the second operational branch to cleanly handle symmetrical decrement calculations: nextX--.

Project: Pac-Man Engine — Open-Corridor State Starvation

Vulnerability: Constraining tracking AI updates exclusively to structural collision handlers.

System Failure: Entering an open horizontal corridor without wall intersections starved the state engine. Lacking boundary collision triggers, the AI became locked in an infinite left-to-right loop, passing vertical corridors completely.

Resolution: Introduced a non-blocking 10% stochastic random weight parameter calculated instantly upon grid alignment coordinates to split tracking vectors organically.

Project: Pac-Man Engine — Over-Engineered Intersection Twitch

Vulnerability: Unbounded mathematical execution checks running at rigid grid boundaries on every frame tick.

System Failure: Caused erratic trajectory jittering as tracking units evaluated reverse directions on back-to-back frames, locking themselves into micro-stutter loops.

Resolution: Normalized coordinate bounds thresholds and implemented the 10% movement variance limits to stabilize entity headings.

Project: The Haunted Waltz 5.0 — Illegal Matrix Index Evaluation

Vulnerability: Incorrect vector variable parsing within the boundary evaluation logic.

if (currentMaze[ghostY][nextX] == 0) // <-- The Bug

System Failure: Evaluated the current row position using horizontal test steps instead of checking the targeted row index. This caused path clipping or threw fatal ArrayIndexOutOfBoundsException crashes near map edges.

Resolution: Re-indexed variables to guarantee explicit verification of the targeted vector space: currentMaze[nextY][ghostX] == 0.

Project: Pac-Man Engine — Multi-System Session Wipe

Vulnerability: Coupling local character positional resets directly to global initialization states.

System Failure: Triggering a collision cleanly deducted life points, but calling the comprehensive resetGame() method accidentally re-initialized session scores to 0 and lives to 3, masking valid terminal game states.

Resolution: Isolated logic blocks by writing a modular resetPositions() method to manipulate coordinate arrays while safeguarding game state primitives.

Project: The Haunted Waltz 5.0 — Synchronous Thread Freezing

Vulnerability: Porting blocking console IO structures into asynchronous graphic event pipelines.

System Failure: Calling sc.nextLine() halted execution of the graphics thread while waiting for manual text entry, starvings display refresh registers and locking the window system UI layer.

Resolution: Completely dropped Scanner logic dependencies and migrated choices into asynchronous modal JOptionPane.showOptionDialog modules.

🤖 Arduino Embedded Systems Failures

Project: Autonomous Robotic Tracking Pet — The "Frozen Brain" Sound Bug

Vulnerability: Running high-latency blocking delay() lines inside computational loops.

System Failure: Calling delays stopped the ATmega328P internal clock cycle. During vocalization routines, the robot completely missed distance tracking pulses, causing it to overshoot targets or drop sensor commands.

Resolution: Removed blocking delays and re-architected all time-based intervals around non-blocking timestamp subtractions running on background millis() counters.

Project: Autonomous Robotic Tracking Pet — Multi-Peripheral Control Interference

Vulnerability: Hardware timer resource collisions across adjacent peripheral communication classes.

System Failure: High-frequency sound production routines clashed with PWM registers assigned to the H-bridge drivers and active tracking servo, leading to severe signal chatter and motor torque dropouts.

Resolution: Remapped internal peripheral channels to guarantee that audio frequencies, PWM speeds, and tracking servos operate on non-conflicting clock registers.

Project: Autonomous Robotic Tracking Pet — Rigid Linear Pathfinding Limits

Vulnerability: Driving physical kinematics via a narrow, fixed 15-degree spatial detection envelope.

System Failure: Moving tracking targets easily breached the narrow sensor array limit, breaking tracking routines and throwing the robot into standby loops.

Resolution: Mounted the transceiver onto an active mechanical servo axis on pin 9, implementing a real-time sweeping radar sweep paired with differential wheel-speed mixers.

🎯 Computer Engineering Synthesis Statement

By diagnosing and adjusting for limitations across hardware and software levels, my engineering methodology expanded from basic scripting to complex Hardware-Software Co-Design.

The core achievement of this sprint was implementing Cross-Platform Adaptive Logic: the data flow patterns generated within my high-level Java structures (such as scaling game speed parameters relative to choice choices) work on the exact same state principles as the low-level firmware clock optimizations written to keep my microcontroller response loops running below a highly optimized 300ms floor.
